# 맛킷리스트 업데이트 소식(버전별 changelog) — 설계 문서

작성일: 2026-06-22
대상: 기존 맛킷리스트 사이트에 "업데이트 소식" 알림 추가 (구현은 추후)

## 한 줄 요약
헤더의 "📢 소식" 버튼을 누르면, 버전별로 분류된 업데이트 내역(아코디언)을 보여주는 모달이
뜬다. 버전 목록은 코드 내 정적 배열로 관리한다(운영자만 작성).

## 확정된 결정
| 항목 | 결정 |
|---|---|
| 위치 | 헤더 "📢 소식" 버튼 → 모달 |
| 분류 | 버전별 아코디언(버전 클릭 → 그 버전 내역 펼침) |
| 새 소식 배지 | 없음(빨간 점 안 씀) |
| 데이터 | 코드 내 정적 배열(별도 DB 없음). 배포 시 항목 추가 |
| 기본 펼침 | 최신 버전만 펼친 상태, 나머지 접힘 |

## 1. 소식 버튼 + 모달
- 헤더에 **📢 소식** 버튼(도움말 버튼 옆). 클릭 시 `#news-modal` 모달.
- 닫기 버튼 + 바깥 클릭으로 닫힘. 모바일 스크롤(`max-height`+`overflow-y`).

## 2. 버전별 아코디언
- 모달에 버전 목록을 최신순으로 표시. 각 항목 헤더 = `버전 + 날짜`.
- 헤더 클릭 → 해당 버전의 변경 내역(불릿 목록) 펼침/접힘 토글.
- 최신(맨 위) 버전은 기본 펼침, 나머지는 접힘.

## 3. 데이터 (정적 배열)
- `mukgit.js`에 `CHANGELOG` 배열:
  ```
  const CHANGELOG = [
    { version: "v1.2.1", date: "2026-06-22", items: ["모바일에서 1위가 맨 위로 오도록 정렬 수정"] },
    { version: "v1.2",   date: "2026-06-22", items: ["내 글 수정 기능 추가(사진·내용)"] },
    { version: "v1.1",   date: "2026-06-22", items: ["구글 로그인", "댓글", "분홍 디자인·떠다니는 배경", "사용법 가이드"] },
    { version: "v1.0",   date: "2026-06-22", items: ["맛집 글쓰기·사진", "추천 랭킹(신전 시상대)"] },
  ];
  ```
- 배열 맨 앞에 새 버전을 추가하면 최신 소식이 위에 표시됨. (버전명·날짜·문구는 구현 때 최종 확정)

## 4. 변경 대상 파일 (구현 시)
- `mukgit.html` — 📢 소식 버튼 + 소식 모달 컨테이너
- `mukgit.css` — 소식 버튼·모달·아코디언 스타일
- `mukgit.js` — `CHANGELOG` 배열, 렌더(아코디언 생성), 열기/닫기, 펼침 토글

## 5. 테스트 (수동 확인, 구현 시)
- 📢 소식 클릭 → 모달에 버전 목록(최신 위). 최신 버전 펼침
- 버전 헤더 클릭 → 내역 펼침/접힘 토글
- 닫기/바깥클릭으로 닫힘, 모바일 스크롤 정상
- 기존 기능·디자인 영향 없음

## 6. 범위 밖(YAGNI)
- 빨간 점 배지/새글 알림, Firestore 동적 소식, 사용자별 읽음 처리, 마크다운 본문

## 상태
- **설계 + 계획만 작성. 구현은 추후(토큰 여유 시) 진행.**
