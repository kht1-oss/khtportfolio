# 이력서용 'AI 역량' 프로젝트 카드 PDF — 설계 문서

작성일: 2026-06-22
대상: 오늘 만든 맛킷리스트 프로젝트를 이력서 'AI 역량' 항목에 바로 쓸 1페이지 PDF로 정리

"AI 도구(Claude Code)를 활용해 기획부터 배포까지 완성한 실서비스"를 강조하는
A4 1페이지 프로젝트 카드를 HTML로 만들고 헤드리스 Chrome으로 PDF 변환한다.

| 공개 여부 | 결과 PDF/HTML은 로컬 전용(`resume/`), 공개 리포에 커밋 안 함 |

## 내용 구성 (1페이지)
1. **헤더** — 프로젝트명 "맛킷리스트(MukgitList)" + 부제 "AI 도구로 기획~배포한 실서비스" + 라이브 URL(`https://kht1-oss.github.io/khtportfolio/mukgit.html`)
2. **프로젝트 개요** — 지인 대상 맛집 공유 웹서비스. 사진과 함께 맛집을 올리고, 추천으로 랭킹(신전 테마), 상세창·댓글.
3. **나의 역할 (AI 활용 방식)** — 핵심. 요구사항 정의·기술 의사결정·검수·배포를 주도. 자연어로 기능을 설계→AI 페어로 구현→검토→반복 개선. (정직한 프레이밍: 직접 손코딩이 아니라 AI를 도구로 활용해 동작·배포되는 결과물 완성)
4. **주요 기능** — 글쓰기(사진 업로드·클라이언트 압축), 추천 랭킹(시상대), 구글 로그인, 계정별 1표, 상세창·댓글, 반응형, 사용법 가이드.
5. **기술 스택** — HTML/CSS/Vanilla JS(ES Modules), Firebase(Authentication·Firestore), GitHub Pages.
6. **문제 해결·성과** — (정직·구체)
   - Firebase Storage 유료화 이슈를 Firestore에 이미지(base64) 직접 저장으로 우회 → 무료 등급 유지
   - 보안 규칙으로 소유권 기반 권한(본인/관리자만 삭제, 계정별 추천) 설계
   - 기획→구현→테스트→배포를 짧은 사이클로 반복, 당일 배포

## 시각/레이아웃
- A4, 단일 컬럼, 여백 충분. 포인트 컬러는 맛킷 테마와 결 맞춰 로즈/골드 절제 사용.
- 섹션 제목 + 불릿. 가독성 우선(인쇄 시 깔끔).
- 하단에 작게 "제작: AI 페어 프로그래밍(Claude Code) 활용 / 2026.06".

## 산출물 & 위치
- `resume/ai-card.html` — 인쇄용 HTML(원본, 수정·재생성 용이)
- `resume/맛킷리스트_AI역량.pdf` — 최종 PDF
- `resume/`는 `.gitignore`에 추가(공개 리포에 미포함). 설계/계획 문서(docs/)만 커밋.

## 생성 방법
- 헤드리스 Chrome: `chrome --headless --print-to-pdf=... --no-margins ai-card.html` (또는 Edge 동일 옵션).
- HTML `@page { size: A4; margin: ... }` 로 1페이지에 맞춤.
