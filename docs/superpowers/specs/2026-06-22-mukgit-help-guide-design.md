# 맛킷리스트 사용법(도움말) — 설계 문서

작성일: 2026-06-22
대상: 기존 맛킷리스트 사이트에 사용법 안내(도움말) 추가

## 한 줄 요약
헤더의 "도움말" 버튼을 누르면, 스크린샷과 함께 5단계 사용법을 보여주는 모달이 뜬다.
스크린샷은 사용자가 `guide/` 폴더에 정해진 이름으로 넣고, 커밋·푸시는 Claude가 한다.

## 확정된 결정
| 항목 | 결정 |
|---|---|
| 화면 소스 | 사용자가 실제 스크린샷 제공 |
| 위치 | 사이트 헤더의 "❓ 도움말" 버튼 → 모달 |
| 노출 | 자동 팝업 아님(필요할 때 클릭) |
| 단계 수 | 5단계 (간단명료) |
| 스크린샷 폴더 | `z:/NewClaude/guide/` (정해진 파일명) |
| 커밋/푸시 | Claude가 수행(사용자는 파일만 저장) |

## 1. 도움말 버튼 + 모달
- 헤더 로그인 영역 옆에 **❓ 도움말** 버튼.
- 클릭 시 모달(`#help-modal`)이 뜨고, 단계가 세로로 나열. 닫기 버튼 + 바깥 클릭으로 닫힘.
- 모바일에서 스크롤 가능(`max-height` + `overflow-y`).
- 맨 위 안내 한 줄: "로그인 없이 구경은 자유, 글·추천·댓글은 로그인 후."

## 2. 단계 구성 (각 단계 = 번호 + 제목 + 한두 줄 설명 + 스크린샷)
1. **구글 로그인** — 우상단 "구글로 로그인" 버튼을 눌러 로그인.
2. **맛집 올리기** — 우하단 ＋ 버튼 → 사진·음식이름·작성자·가격·가게·본문 입력 → "작성".
3. **추천하기** — 카드의 🤍 하트를 누르면 추천(계정당 1번, ❤️로 바뀜).
4. **자세히 보기 & 댓글** — 카드를 누르면 상세창이 열려 큰 사진·본문·댓글 확인, 댓글 작성.
5. **삭제** — 내 글·내 댓글의 🗑️ 로 삭제(관리자는 전체 삭제 가능).

## 3. 스크린샷 운영
- 폴더: `z:/NewClaude/guide/`
- 파일명(고정):
  - `guide/1-login.png`
  - `guide/2-write.png`
  - `guide/3-vote.png`
  - `guide/4-detail.png`
  - `guide/5-delete.png`
- 모달은 각 단계에서 위 이미지를 `<img>`로 불러온다.
- 아직 없는 이미지: `onerror`로 "이미지 준비 중" 자리표시를 보여, **사진이 없어도 텍스트 설명은 작동**.
- 형식 권장: PNG/JPG, 가로 폭 600~1000px 정도면 충분.
- 주의: 스크린샷에 지인 실명/개인정보가 크게 드러나지 않게(사용자 본인 화면 위주).

## 4. 변경 대상 파일
- `mukgit.html` — 도움말 버튼 + 도움말 모달 마크업
- `mukgit.css` — 도움말 버튼·모달·단계·이미지 스타일
- `mukgit.js` — 도움말 모달 열기/닫기, 이미지 onerror 자리표시
- `guide/` — 스크린샷 폴더(빈 폴더는 git이 추적 못 하므로 `guide/README.md`로 폴더 생성·설명)

## 5. 테스트 (수동 확인)
- 도움말 버튼 클릭 → 모달에 5단계 텍스트 표시, 닫기/바깥클릭으로 닫힘
- `guide/`에 이미지가 없을 때 "이미지 준비 중" 자리표시, 있을 때 해당 스크린샷 표시
- 모바일 폭에서 스크롤 정상, 기존 기능·디자인 영향 없음

## 6. 범위 밖(YAGNI)
- 다국어, 동영상 튜토리얼, 자동 첫방문 팝업, 단계별 인터랙티브 투어
