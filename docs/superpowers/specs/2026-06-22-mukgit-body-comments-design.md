# 맛킷리스트 본문 + 댓글 + 떠다니는 배경 — 설계 문서

작성일: 2026-06-22
대상: 기존 맛킷리스트 사이트(로그인 포함)에 글 본문, 댓글, 배경 애니메이션 추가

## 한 줄 요약
카드를 클릭하면 상세창이 열려 큰 사진·본문·댓글을 보여주고, 로그인 사용자가 댓글을
달 수 있게 한다. 배경에는 하트·음식 이모지가 천천히 떠다닌다.

## 확정된 결정
| 항목 | 결정 |
|---|---|
| 본문/댓글 표시 | 카드 클릭 → **상세창(모달)** 에서 표시 |
| 본문 입력 | **선택**(비워도 작성 가능) |
| 댓글 작성 | **로그인 사용자만** |
| 댓글 작성자 이름 | **직접 입력**(별명 가능) |
| 댓글 저장 위치 | 글의 **하위 컬렉션** `posts/{id}/comments` |
| 댓글 삭제 | 댓글 작성자 본인 또는 관리자(`khtc0228@gmail.com`) |
| 배경 | 하트·음식 이모지가 떠다니는 CSS 애니메이션 |

## 1. 상세창(detail modal)
- 랭킹 카드는 현행 유지(작게). **카드의 빈 영역(사진·본문 등)을 클릭하면 상세창**이 뜬다.
- 카드의 하트(추천)·삭제 버튼 클릭은 각자 동작하고 상세창을 열지 않는다(이벤트 구분).
- 상세창 구성: 큰 사진, 음식 이름, 가게·가격, 작성자, **본문**, **댓글 목록 + 댓글 입력**.
- 닫기: 닫기 버튼 또는 바깥 클릭(상세창은 입력 폼이 아니라 바깥 클릭으로 닫혀도 무방).
- 상세창에 표시되는 추천수는 열 때 기준(정적). 추천은 카드에서 한다.

## 2. 본문(body)
- 글쓰기 폼에 **여러 줄 입력칸(textarea)** 추가. 선택 입력(비어도 됨).
- 글 문서에 `body`(string) 필드로 저장. 상세창에 표시(없으면 생략).
- 길이 제한: 1000자.

## 3. 댓글
- 위치: `posts/{postId}/comments/{commentId}` 하위 컬렉션(사진이 큰 글 문서를 무겁게 하지 않기 위함).
- 댓글 필드: `name`(직접 입력 이름), `text`(내용), `commenterUid`(작성자 uid, 권한용·숨김), `createdAt`(시각).
- 작성: **로그인 필요**. 상세창 하단 입력칸(이름 + 내용) → 추가. 내용 길이 500자.
- 표시: 상세창이 열려 있는 동안 해당 글의 댓글을 **실시간 구독**(`onSnapshot`), 작성 시각 오름차순. 상세창 닫으면 구독 해제.
- 삭제: 각 댓글에 작성자 본인 또는 관리자에게만 🗑️ 노출 + 규칙으로 강제.

## 4. 떠다니는 배경
- 콘텐츠 **뒤 배경 레이어**(고정, `position: fixed`, `z-index` 낮게, `pointer-events: none`).
- 하트(🤍❤️🩷)·음식(🍕🍜🍣🍔🍝🥘🍰🍩) 이모지 약 12~16개가 CSS keyframes로 천천히 위로 떠오르며 좌우로 흔들림.
- `@media (prefers-reduced-motion: reduce)`에서는 애니메이션 정지/최소화. 모바일에서는 개수 축소.
- 클릭·스크롤·버튼 동작에 영향 없음(pointer-events 없음).

## 5. 데이터 모델 변경
- `posts` 문서: `body`(string, 선택) 추가. 기존 필드 유지.
- 신규 하위 컬렉션 `posts/{id}/comments`: `name`, `text`, `commenterUid`, `createdAt`.

## 6. 보안 규칙 추가/변경
- `posts` 생성 규칙에 `body`가 있으면 string이어야 함(없어도 허용): `(!('body' in request.resource.data) || request.resource.data.body is string)`.
- 댓글 컬렉션 규칙:
  - 읽기: 누구나
  - 생성: `request.auth != null` + `commenterUid == request.auth.uid` + `name`/`text`가 string + `text` 비어있지 않음
  - 삭제: `request.auth.uid == resource.data.commenterUid` 또는 `request.auth.token.email == 'khtc0228@gmail.com'`
  - 수정: 불허(댓글 수정 없음)

## 7. 순수 함수(테스트 대상)
- `canDeleteComment(comment, user, adminEmail) -> boolean`: 로그인 + (작성자 uid 일치 또는 관리자 이메일).
  (기존 `canDeletePost`와 동일 패턴.)

## 8. 변경 대상 파일
- `mukgit.html` — 글쓰기 폼에 본문 textarea, 상세창(detail) 모달 마크업, 배경 레이어 컨테이너
- `mukgit.css` — 상세창 스타일, 댓글 목록/입력 스타일, 떠다니는 배경 애니메이션
- `mukgit.js` — 상세창 열기/닫기, 본문 저장·표시, 댓글 실시간 구독·작성·삭제, 배경 이모지 생성
- `mukgit.utils.js` — `canDeleteComment` 추가(+테스트)
- `firestore.rules` — body + comments 규칙

## 9. 테스트 (수동 확인)
- 카드 클릭 → 상세창에 사진·정보·본문·댓글. 하트/삭제 버튼은 상세창 안 열고 동작
- 본문 비우고 작성 OK / 본문 적으면 상세창에 표시
- 로그인 후 댓글 작성(이름+내용) → 목록에 즉시 반영. 비로그인은 댓글 입력 막힘/안내
- 내 댓글에만 🗑️ 보이고 삭제 / 관리자 계정은 모든 댓글 🗑️
- 배경 이모지가 떠다니되 클릭·버튼·스크롤 정상
- 기존 기능(로그인/추천/글삭제/디자인) 유지

## 10. 범위 밖(YAGNI)
